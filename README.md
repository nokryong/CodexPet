# CodexPet

Codex 쓸 때 옆에서 놀고 있는 데스크톱 펫입니다. 바탕화면 위를 돌아다니다가 Codex가 일을 시작하면 같이 일하는 척하고, 작업 내용이랑 사용량도 말풍선으로 알려줍니다.

Codex CLI의 펫 에셋(`~/.codex/pets`)을 그대로 가져다 쓰기 때문에, Codex에서 펫을 설치해뒀다면 별도 설정 없이 바로 골라 쓸 수 있습니다.

## 실행

```bash
npm install
npm run start
```

exe로 뽑으려면:

```bash
npm run dist
```

`artifacts/CodexPet-0.1.0.exe` 포터블 실행 파일이 나옵니다. 설치 과정 없이 그냥 실행하면 됩니다. 빌드는 앱을 끈 상태에서 돌려야 합니다(실행 중이면 파일 잠금 때문에 실패).

DevTools가 필요하면 이렇게 켜세요.

```powershell
$env:PET_DEVTOOLS="1"
npm run dev
```

## 뭘 하는 앱인가

### Codex 사용량 확인

펫을 더블클릭하면(우클릭 메뉴에서도 가능) 말풍선으로 사용량이 뜹니다. 5시간 한도와 주간 한도가 각각 몇 % 남았는지, 언제 초기화되는지 보여줍니다. 사용률이 70%를 넘으면 게이지가 노란색, 90%를 넘으면 빨간색이 됩니다.

수치는 Codex가 마지막으로 응답한 시점 기준입니다. Codex를 한동안 안 썼다면 그 사이 회복분은 반영이 안 되는데, 초기화 시각이 이미 지난 기록이면 "이미 초기화됨"으로 표시해줍니다.

사용률이 90%를 넘으면 굳이 확인 안 해도 펫이 알아서 쓰러지면서 경고 말풍선을 띄웁니다. 초기화 주기당 한 번만요.

### Codex 작업 실시간 표시

어느 터미널에서든 Codex가 일을 시작하면 펫이 알아챕니다. `~/.codex/sessions`의 세션 로그를 1.5초 간격으로 tail하는 방식이라 Codex 쪽 설정은 아무것도 안 건드립니다.

- 작업 시작 → 펫이 살펴보기 모션으로 바뀌고 요청 내용이 말풍선에 뜸
- 작업 중 → Codex의 중간 메시지, 파일 수정, 웹 검색 내용이 실시간으로 갱신됨
- 작업 완료 → 폴짝 뛰고 마지막 메시지를 보여준 뒤 다시 놀러 다님
- 작업 중단 → 쓰러짐

세션 여러 개를 동시에 돌려도 각각 추적하고, Codex가 죽어서 완료 이벤트가 안 와도 5분 뒤 알아서 원래 상태로 돌아옵니다.

### 펫 바꾸기

우클릭 → "펫 바꾸기"에서 고르면 즉시 바뀌고, 선택은 다음 실행 때도 유지됩니다. 목록에 나오는 순서는:

1. exe 옆 `pet/spritesheet.webp` — 직접 만든 스프라이트를 쓰고 싶을 때
2. `~/.codex/pets`에 설치된 펫들 — Codex에서 펫을 추가하면 여기에도 자동으로 나타남
3. 내장 기본 펫

## 조작법

| 동작 | 반응 |
|---|---|
| 클릭 | 인사 |
| 더블클릭 | 점프 + Codex 사용량 표시 |
| 드래그 | 창 이동 |
| 우클릭 | 메뉴 (사용량, 펫 바꾸기, 일시정지, 마우스 따라가기, 자동 실행, 숨기기 등) |
| 시스템 트레이 | 보이기, 숨기기, 펫 바꾸기, 완전 종료 |
| 말풍선 클릭 | 닫기 |

우클릭 메뉴의 "숨기기"는 창만 감추고 앱은 시스템 트레이에 남깁니다. 완전히 끄려면 시스템 트레이 아이콘을 우클릭해서 "완전 종료"를 누르면 됩니다.

우클릭 메뉴의 "윈도우 시작 시 자동 실행"을 켜면 로그인할 때 같이 뜹니다.

## 커스텀 스프라이트 만들기

Codex 펫 스프라이트 규격을 그대로 따릅니다.

- 전체 크기 1536x1872, 셀 192x208의 8열 x 9행 그리드
- row가 상태, column이 프레임

| row | 상태 | 프레임 수 |
|---|---|---|
| 0 | idle | 6 |
| 1 | runningRight | 8 |
| 2 | runningLeft | 8 |
| 3 | waving | 4 |
| 4 | jumping | 5 |
| 5 | failed | 8 |
| 6 | waiting | 8 |
| 7 | running | 8 |
| 8 | review | 8 |

이 규격으로 만든 `spritesheet.webp`를 exe 옆 `pet/` 폴더에 넣으면 메뉴에 "커스텀"으로 나타납니다.

## 코드 구조

- `src/main.js` — 창 관리, 이동 로직, 메뉴, 말풍선 제어. 이동 속도나 말풍선 크기 같은 값은 상단의 `MOVEMENT_CONFIG`, `BUBBLE_CONFIG`에 모여 있음
- `src/codex-watcher.js` — Codex 세션 로그 감시. 한도 경고 기준은 `main.js`의 `USAGE_WARN_THRESHOLD_PERCENT`
- `src/renderer.js` — 스프라이트 애니메이션 재생. 상태 정의는 `PET_STATES`
- `src/bubble.html` / `bubble.js` — 말풍선 창

아직 안 한 것: 자동 업데이트.
